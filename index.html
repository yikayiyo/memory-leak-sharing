什么是内存泄漏？

定义

内存泄漏 = 程序中已动态分配的堆内存由于某种原因未释放或无法释放，造成系统内存的浪费，导致程序运行速度减慢甚至系统崩溃。

为什么前端要注意？

传统认知：前端页面刷新 = 内存自动清理

现实情况：
• SPA（单页应用）长时间不刷新
• 用户停留时间长（表格、编辑器类应用）
• 移动端内存有限
影响严重程度

轻度： 页面变慢、卡顿

中度： 浏览器崩溃、白屏

重度： 整个系统资源耗尽

场景1：未清理的定时器

❌ 问题代码 
// React 组件
const MyComponent = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // ❌ 每秒更新一次，但组件卸载时没有清理
    const timer = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);
    // 缺少 return 清理函数！
  }, []);

  return <div>Count: {count}</div>;
};
✅ 正确写法 
const MyComponent = () => {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const timer = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);

    // ✅ 清理定时器
    return () => clearInterval(timer);
  }, []);

  return <div>Count: {count}</div>;
};
有 setInterval 就必须有 clearInterval，有 setTimeout 就要有 clearTimeout

场景2：未清理的事件监听器

❌ 问题代码 
class MyComponent extends React.Component {
  componentDidMount() {
    // ❌ 全局事件监听，但从未移除
    window.addEventListener('resize', this.handleResize);
    document.addEventListener('click', this.handleClick);
  }

  render() { /* ... */ }
}
✅ 正确写法 
class MyComponent extends React.Component {
  componentDidMount() {
    window.addEventListener('resize', this.handleResize);
    document.addEventListener('click', this.handleClick);
  }

  componentWillUnmount() {
    // ✅ 移除所有事件监听器
    window.removeEventListener('resize', this.handleResize);
    document.removeEventListener('click', this.handleClick);
  }

  render() { /* ... */ }
}

🎣 Hooks 版本

const useResize = (callback) => {
  useEffect(() => {
    window.addEventListener('resize', callback);
    return () => window.removeEventListener('resize', callback);
  }, [callback]);
};

场景3：闭包中的循环引用

❌ 问题代码 
function createLeak() {
  const largeArray = new Array(1000000).fill({ data: 'big' });

  const handler = function() {
    // ❌ 闭包引用了 largeArray，handler 无法被 GC
    console.log(largeArray.length);
  };

  document.getElementById('button').addEventListener('click', handler);
}
✅ 正确写法 
function createNoLeak() {
  const largeArray = new Array(1000000).fill({ data: 'big' });
  const length = largeArray.length; // 只保留需要的值

  const handler = function() {
    // ✅ 闭包只引用基本类型，largeArray 可以被 GC
    console.log(length);
  };

  const btn = document.getElementById('button');
  btn.addEventListener('click', handler);

  // 及时移除
  setTimeout(() => {
    btn.removeEventListener('click', handler);
  }, 5000);
}

场景4：未清理的第三方库实例

⚠️ 常见需要清理的库

库名 清理方法
Chart.js chart.destroy()
ECharts chart.dispose()
Three.js scene.dispose(), geometry.dispose()
Mapbox map.remove()
Leaflet map.remove()
Video.js player.dispose()
✅ Chart.js 示例 
const ChartComponent = () => {
  const chartRef = useRef(null);
  const chartInstance = useRef(null);

  useEffect(() => {
    if (chartInstance.current) {
      chartInstance.current.destroy();
    }

    chartInstance.current = new Chart(chartRef.current, config);

    // ✅ 清理函数
    return () => {
      if (chartInstance.current) {
        chartInstance.current.destroy();
      }
    };
  }, [chartData]);

  return <canvas ref={chartRef} />;
};

如何检测内存泄漏

🔍 Heap Snapshot（堆快照）

  1. 打开 Chrome DevTools → Memory
  2. 选择 "Heap snapshot"
  3. 拍摄第一张快照（基准）
  4. 执行操作（打开/关闭页面、触发交互）
  5. 拍摄第二张快照
  6. 选择第二个快照 → "Comparison" 视图
  7. 查看 "+" 号（新增的对象）

📈 Allocation Timeline（分配时间线）

  1. 选择 "Allocation instrumentation on timeline"
  2. 开始录制
  3. 操作页面
  4. 停止录制
  5. 分析蓝色柱状图（分配的对象）

📊 Allocation Sampling（采样分析）

性能开销小，适合长时间监控，不显示完整对象，只显示调用栈。

🧪 快速测试方法

let iterations = 0;
const maxIterations = 20;

function testMemory() {
  if (iterations >= maxIterations) {
    console.log('Test complete. Check if memory keeps growing.');
    return;
  }

  // 执行可能泄漏的操作
  const btn = document.querySelector('#leak-button');
  btn.click();

  iterations++;
  console.log(`Iteration ${iterations}`);

  setTimeout(testMemory, 1000);
}

testMemory();

关键指标识别

📊 正常情况

  • ✅ 内存有波动，但会回落
  • ✅ 峰值和基线保持稳定

🚨 内存泄漏特征

  • ❌ 内存持续增长（锯齿状上升）
  • ❌ 回收后基线不断抬高
  • ❌ DOM Nodes 数量持续增加
  • ❌ Event Listeners 数量不降反升

最佳实践与工具

✅ React Hooks 清理清单

useEffect(() => {
  // 1. 设置定时器/轮询
  const timer = setInterval(callback, 1000);

  // 2. 添加事件监听
  window.addEventListener('resize', handleResize);

  // 3. 订阅数据流
  const subscription = observable.subscribe(observer);

  // 4. 创建第三方库实例
  const chart = new Chart(canvas, config);

  // ✅ 返回清理函数
  return () => {
    clearInterval(timer);
    window.removeEventListener('resize', handleResize);
    subscription.unsubscribe();
    chart.destroy();
  };
}, [dependencies]);

总结

🎯 核心原则

  1. 成对出现： 创建就要有销毁
  2. 及时清理： 组件卸载时必须清理所有资源
  3. 最小引用： 闭包只保留必要的数据
  4. 定期检测： 开发阶段就要关注内存指标

✅ 记住一句话

"每一个 useEffect，都要问自己：组件卸载时会发生什么？"

Q&A

Q: 内存泄漏和性能差是一回事吗？

A: 不是。内存泄漏是"内存用不掉"，性能差可能是"计算量大"。但长期内存泄漏会导致性能恶化。

Q: React 函数组件不需要清理吗？

A: 一样需要！useEffect 的 cleanup function 就是干这个的。

Q: 多少内存算泄漏？

A: 没有固定标准。关键是趋势——如果操作后内存一直涨不下来，就有问题。
